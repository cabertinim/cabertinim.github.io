6.1 Fudamentos e aplicações

6.1.1 Hipótese estatística

Os testes de hipóteses oferecem uma metodologia para avaliar se os dados amostrais fornecem evidências suficientes para apoiar ou rejeitar uma hipótese estatística formulada.

Essa hipótese estatística consiste em afirmações sobre um parâmetro populacional de interesse, que devem ser estabelecidas antes da análise dos dados:

  • Hipótese nula(\(H_0\)): é a hipótese (afirmação) que estamos colocando à prova, geralmente contendo uma condição de igualdade ou de não efeito.

  • Hipótese alternativa(\(H_1\) ou \(H_a\)): é a hipótese complementar, que será considerada aceitável caso \(H_0\) seja rejeitada. Geralmente, a hipótese alternativa representa o que o pesquisador gostaria de comprovar.

6.1.2 Erros nos testes de hipóteses

Em um teste de hipótese, dois erros podem ocorrer:

  • Erro tipo I: rejeitar \(H_0\) quando ela é verdadeira;
  • Erro tipo II: não rejeitar \(H_0\) quando ela é falsa.

As probabilidades de ocorrência do erro tipo I e do erro tipo II são denominadas, respectivamente, de \(\alpha\) e \(\beta\):

\[ \alpha = P\left( Erro \ Tipo \ I \right) = P\left( rejeitar \ H_0 | H_0 \ é \ verdadeira \right) \] \[ \beta = P\left( Erro \ Tipo \ II \right) = P\left( não \ rejeitar \ H_0 | H_0 \ é \ falsa \right) \]

Important

Em um teste de hipóteses, a utilização de dados amostrais implica a possibilidade de erros de decisão, pois o verdadeiro valor do parâmetro populacional é desconhecido.

1 Ilustração do erro tipo I

2 Ilustração do erro tipo II

Essas duas probabilidades, \(\alpha\) e \(\beta\), são estabelecidas antes da análise. \(\alpha\) é chamado de nível de significância.

Em estudos na área da Saúde, frequentemente assume-se \(\alpha = 5\%\) e \(\beta = 20\%\).

Vale destacar que, ao reduzir a probabilidade de cometer um tipo de erro, aumenta-se a probabilidade de cometer o outro. Entretanto, o erro do tipo I em muitos contextos é considerado como o mais grave. Também, por ser mais controlável, é aquele que se busca minimizar prioritariamente.

Um exemplo

Em um teste de medicamentos, um erro tipo I significaria aprovar um medicamento ineficaz, o que pode causar danos à saúde pública. Já um erro tipo II é uma falha em detectar um verdadeiro efeito do medicamento, que geralmente é menos crítico, pois apenas implica que mais dados ou estudos podem ser necessários.

6.1.3 Utilidade dos testes estatísticos

Os testes de hipóteses estatísticas são ferramentas fundamentais para verificar a probabilidade de um evento ocorrer ao acaso, sendo esta probabilidade representada pelo p-valor.

A hipótese nula (\(H_0\)) é rejeitada quando o p-valor for menor do que o nível de significância (\(\alpha\)) previamente estabelecido.

Warning

Embora os testes estatísticos não eliminem completamente a possibilidade de erro, eles reduzem as incertezas, pois o p-valor indica a probabilidade de se observar um valor da estatística de teste igual ou mais extremo do que o observado na amostra sob análise, assumindo que a hipótese nula (\(H_0\)) seja verdadeira.

6.2 Teste de independência entre duas variáveis qualitativas

Para cada situação a ser analisada, existe um teste estatístico apropriado. Quando o objetivo é verificar a associação entre duas variáveis qualitativas (categóricas), os testes mais comumente utilizados são:

  • Teste Qui-quadrado de Pearson para a independência, incluindo suas variações;

  • Teste Exato de Fisher.

6.2.1 Teste Qui-quadrado de Pearson para a independência

O teste Qui-quadrado de Pearson é aplicado para verificar se existe associação entre duas variáveis qualitativas. Contudo, sua utilização depende de algumas condições:

  1. A amostra deve ser aleatória e ter mais de vinte elementos*;

  2. As frequências esperadas devem ser maiores ou iguais a 5.

Se essas condições não forem atendidas, recomenda-se o uso de outros métodos, como:

  • Teste Exato de Fisher, para pequenos tamanhos amostrais e tabelas 2x2;

  • Teste Qui-quadrado via Markov Chain Monte Carlo (MCMC), para maior flexibilidade computacional.

6.2.1.1 Exemplo prático:

Um dos objetivos do problema dado em surto de intoxicação alimentar estafilocócica, banco de dados surto, é verificar quais alimentos estão associados aos casos de intoxicação alimentar. Logo, uma das hipóteses investigadas é: “O consumo de salada está associado aos casos de intoxicação alimentar?”

Resolução:

Code
#Carregamento dos pacotes necessários
library(readxl)
library(tidyverse)
library(knitr)
library(gtsummary)
options(scipen = 999) # para evitar números em notação científica
options(OutDec = ",") # definir separador decimal

#Importação do banco de dados
surto <- read_excel("./data/surto.xlsx", sheet = "Plan1") 

#Limpeza/tratamento das variáveis
surto <- mutate(surto, 
caso = factor(caso, labels=c('Não','Sim')),
sexo = factor(sexo, labels=c("Feminino","Masculino")),
ocupa = factor(ocupa),
diarreia = factor(diarreia, labels=c('Não','Sim')),
colica = factor(colica, labels=c('Não','Sim')),
vomito = factor(vomito, labels=c('Não','Sim')),
nausea = factor(nausea, labels=c('Não','Sim')),
cefaleia = factor(cefaleia, labels=c('Não','Sim')),
desidratacao = factor(desidratacao,labels=c('Não','Sim')),
febre = factor(febre,labels=c('Não','Sim')),
servsaude = factor(servsaude, labels=c('Não','Sim')),
gravidade = factor(gravidade, order = TRUE, labels=c('Leve','Moderado',"Grave")),
salada = factor(salada, labels=c('Não','Sim')),
pao = factor(pao, labels=c('Não','Sim')),
medalhao = factor(medalhao, labels=c('Não','Sim')),

pudim = factor(pudim, labels=c('Não','Sim')),
frutas = factor(frutas, labels=c('Não','Sim')),
agua = factor(agua, labels=c('Não','Sim')),
suco = factor(suco, labels=c('Não','Sim'))
  )

surto$leuco[123] <- 6980

#criando variável idade
surto <- surto %>% 
    mutate(dtnasc = as_date(dtnasc),
           idade = 2018 - year(dtnasc) )

Antes da análise, devem ser estabelecidos as hipóteses nula e alternativa e o nível de significância:

  • \(H_0\): a salada não está associada à intoxicação alimentar.

  • \(H_1\): a salada está associada à intoxicação alimentar.

  • \(\alpha = 5\%\)

Depois, é preciso verificar se as condições de realização do teste estão satisfeitas. Isto é feito analisando-se a tabela de valores esperados, retornada pela função chisq.test( ):

Code
#Teste Qui-quadrado
Teste1 <- chisq.test(surto$salada,surto$caso, 
                     correct = FALSE)

#verificando se as frequências esperadas são >= a 5.
kable(Teste1$expected ) #frequências esperadas

Não 44,66667 22,33333
Sim 55,33333 27,66667

As condições do teste foram atendidas: a amostra possui tamanho maior do 20 e todas as frequências esperadas são maiores ou iguais a 5.

Resultado do teste:

Code
Teste1 # resultado do teste qui-quadrado

    Pearson's Chi-squared test

data:  surto$salada and surto$caso

Interpretação: Como o p-valor obtido é 0,353, superior a 5%, não rejeitamos \(H_{0}\). Logo, o consumo de salada no almoço não está associada ao adoecimento dos participantes, ao nível de significância de 5%.

Outra solução utilizando-se o pacote gtsummary:

Code
surto %>% 
  select(salada, caso) %>% 
  tbl_summary(
    by=caso,
    label = salada ~ "Comsumiu salada no almoço?",
    ) %>% 
  add_p(
    test = everything() ~ "chisq.test.no.correct",
    pvalue_fun = label_style_pvalue(digits = 3)
    ) %>% 
  modify_spanning_header(c(stat_1, stat_2) ~ "**Intoxicação alimentar?**") %>%

6.2.2 Teste Exato de Fisher

É utilizado na verificação de associação entre duas variáveis categóricas binárias (tabelas de contingência 2x2). Pode ser uma alternativa ao teste Qui-quadrado de Pearson para a independência quando as frequências esperadas são menores do que 5 e a amostra é pequena.

Code
#Teste Exato de Fisher
Teste <- fisher.test(surto$salada,
                     surto$caso,
                     conf.level= 0.95) #nível de confiança utilizado (1-alpha).

Se houver mais de duas categorias em pelo menos uma das variáveis nominais e as condições para o teste Qui-quadrado de Pearson não forem atendidas, pode-se utilizar o argumento `simulate.p.value = TRUE` na função `chisq.test()`, o que permite a aplicação de um método baseado em simulação Monte Carlo (MCMC).
:::

### Teste Exato de Fisher

É utilizado na verificação de associação entre duas variáveis categóricas binárias (tabelas de contingência 2x2). Pode ser uma alternativa ao teste Qui-quadrado de Pearson para a independência quando as frequências esperadas são menores do que 5 e a amostra é pequena.

```{r, Fisher}
#Teste Exato de Fisher
Teste <- fisher.test(surto$salada,
                     surto$caso,
                     conf.level= 0.95) #nível de confiança utilizado (1-alpha).

Teste # chama a resposta do teste
```

*Interpretação:* A inferência permanece a mesma, não rejeitamos $H_{0}$. O consumo de salada não está associado ao adoecimento dos participantes, ao nível de significância de 5%.

Usando o pacote `gtsummary`, temos:

```{r}
surto %>% 
  select(salada, caso) %>% 
  tbl_summary(
    by=caso,
    label = salada ~ "Comsumiu salada no almoço?",
    ) %>% 
  add_p(
    test = everything() ~ "fisher.test",
    pvalue_fun = label_style_pvalue(digits = 3)
    ) %>% 
  modify_spanning_header(c(stat_1, stat_2) ~ "**Intoxicação alimentar?**") %>% 
  modify_header(label ~ "**Variável**", p.value ~ "*P*-valor") %>% 
  modify_footnote(p.value ~ "Teste exato de Fisher")
```
