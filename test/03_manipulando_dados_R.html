Ler e Manipular dados pode não ser tarefa fácil, mas nesse tutorial vamos simplificar tudo para que você possa entender melhor!

3.1 Instalando pacotes

Para trabalhar com leitura e manipulação de dados, você deverá instalar um pacote do RStudio chamado tidyverse.

O pacote tidyverse é uma coleção de oito pacotes R construídos para operarem juntos. Esse pacote contém inúmeros pacotes importantes para manipular dados, inclusive o pacote dplyr, muito utilizado para transformar e resumir dados tabulares com linhas e colunas.

Code
#Código para instalar o pacote tidyverse:
install.packages("tidyverse")

3.2 dplyr

Considerado o pacote mais importante do tidyverse

3.2.1 O que é o dplyr?

O dplyr é um poderoso pacote R para transformar e resumir dados tabulares com linhas e colunas. Para maiores informações, pode-se consultar https://dplyr.tidyverse.org/.

3.2.2 Qual sua utilidade?

Contém funções que permite a execução de operações comuns de manipulação de dados, como filtrar linhas, selecionar colunas específicas, reordenar linhas, adicionar novas colunas e resumir dados. Além de, dispor de uma função útil para concatenar mais de um tibble/data.frame.

3.2.3 Qual vantagem em seu uso?

Comparadas às funções básicas em R como split(), subset(), apply(), sapply(), lapply(), tapply() and aggregate(), as funções no dplyr são mais fáceis de trabalhar, mais consistentes na sintaxe e melhor direcionadas para análise de dados em cima de um data.frame, ao invés de trabalhar apenas vetores.

3.2.4 Funções importantes do pacote dplyr

3.2.5 Importando a base de dados

Podemos importar 2 tipos de bases de dados:

  1. Base de dados tabulares: arquivos em .csv, .tsv, .fwf.Para importar esse tipo de base, usamos os comandos:

  • read_csv("file.csv") >> colunas delimitadas por vírgulas

  • read_csv2("file2.csv") >> colunas delimitadas por ponto e vírgula

  1. Base de dados em arquivos Excel: arquivos em .xls, .xlsx.Para importar esse tipo de base, usamos os comandos:
  • read_excel("file.xls") ou read_excel("file.xlsx") >> depende de qual extensão está salvo o arquivo.

Para exemplificação, considere o problema de surto de intoxicação alimentar estafilocócica ocorrido durante evento científico (disponível aqui); base de dados surto.

#Lendo a base dados utilizando a função read_excel() do pacote readxl

Nota: podemos automatizar o processo acima usando a função mutate_at(). A função mutate_at() permite modificar várias colunas ao mesmo tempo, aplicando uma transformação específica a essas colunas. surto_factor <- surto %>% 
  mutate_at( c("caso","sexo","ocupa", "diarreia","colica", "vomito", "nausea", "cefaleia", "desidratacao", "febre", "servsaude", "gravidade", "salada", "pao", "medalhao", "macarrao", "molhoqueijo", "molhoverm", "pudim", "frutas", "agua", "suco"), as.factor )

3.2.12 Sumarizando uma coluna/variável (summarise())

A função summarise() resume as variáveis de um data.frame. Ela aplica funções de agregação a uma ou mais colunas e retorna uma nova tabela com esses resumos.

Funções comuns usadas com summarise():

  • n() - conta o número de observações em cada grupo.

  • mean(), sum(), min(), max() - calculam a média, soma, mínimo, máximo, respectivamente.

  • sd(), var() - calculam o desvio padrão e a variância.

Imagine que nosso interesse seja resumir a variável leuco, através do cálculo da média e do desvio padrão.

Code
surto_modif %>% 
  summarise(media = mean(leuco, na.rm = TRUE),
            dp = sd(leuco, na.rm = TRUE)
            )

As funções `left_join()`, `right_join()`, `full_join` e `inner_join()` do `dplyr` são utilizadas para unir duas tabelas (ou data.frame), baseando-se em colunas comuns (chamadas de "chaves").

![Representação das funções "join" do dplyr](./figures/join.jpeg)

-   `left_join()`

A função `left_join()` retorna **todas as linhas da tabela da esquerda** (a primeira tabela), e adiciona as colunas correspondentes da tabela da direita (segunda tabela). Se não houver correspondência, o valor da coluna da direita será preenchido com `NA`.

Para exemplificarmos a função `left_join()` considere o exemplo abaixo.

```{r}
# Criando uma base de dados com os indivíduos 2, 4, 5 e 6 e as variáveis id, caso e sexo 
df1 <- surto_modif %>% 
  select(id, caso, sexo) %>% 
  filter(id %in% c(2,4,5,6)) 

# Criando uma base de dados com os indivíduos 2, 3, 4 e 5 e as variáveis id e ocupa
df2 <- surto_modif %>% 
  select(id, ocupa) %>% 
  filter(id %in% c(2,3,4,5))

# juntando as bases com left_join()
df_combinada <- df1 %>%
  left_join(df2, by = "id")

# Visualizando
df1
df2
df_combinada
```

-   `right_join()`

Ao contrário do `left_join()`, o `right_join()` mantém **todas as linhas da tabela da direita** e adiciona as colunas correspondentes da tabela da esquerda. Se não houver correspondência nas colunas da tabela da esquerda, o valor será preenchido com `NA.`

```{r}
# juntando as bases com right_join()
df_combinada_2 <- df1 %>%
  right_join(df2, by = "id")

# Visualizando
df1
df2
df_combinada_2
```

-   `full_join()`

A função `full_join()` do pacote `dplyr` combina duas tabelas (`data.frames`) usando uma ou mais colunas em comum, retornando **todas as linhas de ambas as tabelas**. Quando não há correspondência entre os dados nas duas tabelas, a função preenche os valores com `NA`.

```{r}
# juntando as bases com full_join()
df_combinada_3 <- df1 %>%
  full_join(df2, by = "id")

# Visualizando
df1
df2
df_combinada_3
```

Outro exemplo usando `full_join()` :

```{r}
# Criando uma base de dados com os indivíduos 1, 2 e 3 e as variáveis id, caso e sexo 
ex1 <- surto_modif %>% 
  select(id, caso, sexo) %>% 
  filter(id %in% 1:3) 

# Criando uma base de dados com os indivíduos 4 e 5 e as variáveis id, sexo e ocupa
ex2 <- surto_modif %>% 
  select(id, sexo, ocupa) %>% 
  filter(id %in% 4:5)

# Unindo as bases pelas chaves id e sexo
ex_full <- ex1 %>%
  full_join(ex2, by = c("id", "sexo"))

ex1
ex2
ex_full
```

-   `inner_join()`

A função `inner_join()` só retorna as linhas que têm correspondência nas duas tabelas, ou seja, apenas os valores que aparecem nas duas tabelas serão mantidos.

```{r}
# juntando as bases com inner_join()
df_combinada_4 <- df1 %>%
  inner_join(df2, by = "id")

# Visualizando
df1
df2
df_combinada_4
```
